Intro to Workflow Manager

ArcGIS Workflow Manager for Server is an extension of ArcGIS Server that provides a framework to organize, centralize, and standardize project workflows using web services. The Workflow Manager tasks have methods to extend Workflow Manager functionality to web applications.

ArcGIS Workflow Manager for Server provides you the ability to do the following:

  • Query and find jobs - Query jobs based on id of an existing query or by using an ad-hoc query definition. Find jobs using a freeform text search in the system. The jobs returned can be used to keep a track of work in the system or execute work assigned.

  • Create jobs - Create jobs based on a specific parameters such as assignment, dates, associated location of interest, priority and more.

  • Close and delete jobs - Close and delete jobs as needed once the work has been completed.

  • Update job properties - Update job properties to change the assignment, dates, priority and description for the job.

  • Execute jobs - Execute step and mark a step as completed to work through the steps in the workflow. Recreate workflows for the jobs whose workflow was completed but needs to be reactivated.

  • Manage history - Log an activity in history for the job and get the activity log for a job.

  • Extended properties - Display extended properties related to a job and edit them as needed.

  • Attachments - Display attachments related to a job and add embedded and linked attachments to the job.

  • Manage Holds - Display holds on a job which suspend the job. Add and release holds on the job.

  • Execute reports - Display reports for viewing the overall job information based on a criteria defined in the report. Get information about all reports and get raw report data.
